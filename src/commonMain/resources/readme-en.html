Madura - why yet another programming language?

Madura is the name of my pet programming language, my playground for experimenting with new technologies, my laboratory to try out new ideas. Madura is also the name of a tiny island near Java, which reflects the initial intention quite well: At first, Madura wasn't much more than a kind of Java dialect, my attempt to design a "better" Java. The first draft came from a time when Java 8 was the latest version, when I was already somewhat familiar with Spring Boot and Project Lombok and just before I got to know Kotlin. I do not recall every adjustment I had planned, but some of the major changes include:

  • Compiler hooks can be implemented directly in the source code as part of the project.
  • To make a class abstract, the bodies of some its method's bodies are omitted.
  • Classes and methods are public by default, while attributes are private.
  • The keyword shared is used instead of static and the type none instead of void.
  • Method parameters are always final.
  • Checked exceptions are removed.
  • An optional array parameter in main method and a shorter println method.
