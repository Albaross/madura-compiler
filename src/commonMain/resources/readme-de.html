Madura - warum noch eine Programmiersprache?

Madura ist der Name meiner Hobby-Programmiersprache, meine Spielwiese zum Experimentieren mit neuen Technologien, mein Labor um neue Ideen auszuprobieren. Madura ist auch der Name einer winzigen Insel in der Nähe von Java, was die ursprüngliche Intention recht gut widerspiegelt: Zunächst war Madura nicht viel mehr als eine Art Java-Dialekt, mein Versuch, ein "besseres" Java zu konzipieren. Der erste Entwurf stammt aus einer Zeit, in der Java 8 die neueste Version war, als ich mit Spring Boot und Project Lombok bereits einigermaßen vertraut war und kurz bevor ich Kotlin kennenlernte. Ich erinnere mich nicht mehr an jede geplante Anpassung, aber einige der wichtigsten Änderungen sind:

  • Compiler-Hooks können als Teil des Projekts direkt im Quellcode implementiert werden.
  • Um eine Klasse abstract zu machen, werden die Bodies einiger ihrer Methoden weggelassen.
  • Klassen und Methoden sind standardmäßig public, während Attribute private sind.
  • Das Keyword shared wird anstelle von static und der Typ none anstelle von void verwendet.
  • Methodenparameter sind immer final.
  • Checked Exceptions werden entfernt.
  • Optionale Array-Parameter in der main-Methode und eine kürzere println-Methode.
A short program might look like this: @Data class Person { final String name; } class Main { shared none main() { final Person person = new Person("Tom"); println("Hello " + person.getName()); } }
